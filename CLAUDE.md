# ServiceVault — Claude Code Guide

## What This Project Is

ServiceVault is a **mobile-first PWA** for small field service businesses (lawn care, landscaping, etc.). It runs entirely in the browser with **no backend** — all data lives in IndexedDB via Dexie.js. Deployed on Vercel.

Live: https://servicevault-eta.vercel.app

---

## Tech Stack

- **React 19** + TypeScript + Vite 6
- **Tailwind CSS v4** (no config file — uses `@tailwindcss/vite` plugin)
- **shadcn/ui** components (`components.json` at root)
- **Dexie.js** — IndexedDB ORM, schema in `src/db/index.ts`
- **Zustand** — lightweight state stores in `src/store/`
- **React Router v6** — all routes lazy-loaded in `src/App.tsx`
- **@dnd-kit** — drag-and-drop for routes and lists
- **vite-plugin-pwa** — service worker + manifest

## Database Schema (Dexie v4 — src/db/index.ts)

All tables use IndexedDB via Dexie. The DB class is `ServiceVaultDB`, exported as `db`.

| Table | Key fields | Notes |
|---|---|---|
| `clients` | id, name, status, tags | status: active / quote / inactive |
| `properties` | id, clientId, address | one client → many properties |
| `serviceItems` | id, name, category, active | seeded on first launch |
| `jobs` | id, clientId, propertyId, status, scheduledDate | status: scheduled/in_progress/completed/skipped/cancelled |
| `jobLineItems` | id, jobId, serviceItemId | line items on a job |
| `invoices` | id, jobId, clientId, number, status | status: draft/sent/paid/overdue/void |
| `payments` | id, invoiceId, amount, method | |
| `recurringSchedules` | id, clientId, rrule | rrule string for recurrence |
| `routeStops` | id, routeDate, jobId, position | daily route ordering |
| `jobPhotos` | id, jobId, type, data (Blob) | before/after photos |
| `clientLists` | id, name | custom named lists |
| `clientListMembers` | id, listId, propertyId | many-to-many |
| `savedRoutes` | id, name, propertyIds | |
| `settings` | key, value | key-value store; `onboardingDone` flag |

Current DB version: **8**. Always increment the version when adding/changing indexes.

## Key Patterns

### Reading data
Use `useLiveQuery` from `dexie-react-hooks` for reactive queries:
```ts
import { useLiveQuery } from "dexie-react-hooks";
const clients = useLiveQuery(() => db.clients.toArray());
```

### Writing data
Direct Dexie calls — no abstraction layer:
```ts
await db.clients.add({ ... });
await db.clients.update(id, { status: "inactive" });
await db.clients.delete(id);
```

### Routing
All routes are in `src/App.tsx`. Adding a new page = create `src/pages/NewPage.tsx` + add a lazy import + add a `<Route>` entry.

### Styling
Use Tailwind utility classes. shadcn/ui components are in `src/components/ui/`. Add new shadcn components with `npx shadcn@latest add <component>`.

### State (Zustand)
Prefer `useLiveQuery` for DB-backed state. Zustand stores are for UI state only (e.g., selected date, filter values).

## Onboarding Flow

- On first load, `OnboardingGuard` checks `db.settings.get("onboardingDone")`
- If not set, redirects to `/onboarding`
- Onboarding sets `db.settings.put({ key: "onboardingDone", value: "true" })`

## Common Tasks

**Add a new page:**
1. Create `src/pages/MyPage.tsx`
2. In `App.tsx`: add `const MyPage = React.lazy(() => import("@/pages/MyPage"))`
3. In `App.tsx`: add `<Route path="my-path" element={<MyPage />} />`

**Add a new DB table:**
1. Add interface in `src/db/index.ts`
2. Add `EntityTable<T, "id">` field to `ServiceVaultDB`
3. Increment DB version and add `.stores()` entry

**Add a shadcn component:**
```bash
npx shadcn@latest add button
```

**Export all data:** Use the existing `ExportDataPage` — it serialises all tables to JSON.

## Recent Changes (April 2026)

### Client form — name-only creation
- `ClientFormPage.tsx`: First property name auto-syncs to `{ClientName}'s Property` as the user types the client name
- Auto-sync stops if the user manually edits the property name (`nameTouched` flag on `PropertyForm`)
- On save, the first property is **always** created even if address is blank — no longer required
- Edit mode: existing property names are loaded with `nameTouched: true` so they never get overwritten

### Saved Routes page (`/more/saved-routes`)
- New page: `src/pages/SavedRoutesPage.tsx`
- Listed under More → Saved Routes (between Saved Lists and Service Templates)
- Shows all saved routes with **Load** (adds to today's route + navigates to Route tab) and **Delete**
- Save route already existed in `RoutePage.tsx` via the ⋮ menu → "Save Route" (only visible when stops are on the route)

### Fix: "Add to..." button in client selection mode
- `ClientsPage.tsx`: Button was broken due to a race condition — `selectedPropertyIds` (a `useLiveQuery`) could still be `[]` when the button was clicked
- Fixed by resolving property IDs **imperatively** (`handleAddTo` async function) instead of relying on a reactive query
- Button is now **disabled** (greyed out) until at least one client is selected
- Picker closes and clears selection + selection mode on dismiss

### React Router v6 future flag warnings silenced
- `App.tsx` `BrowserRouter` now has `future={{ v7_startTransition: true, v7_relativeSplatPath: true }}`

### Known gotchas
- `useLiveQuery` returns `undefined` on first render — always use `?? []` or `?? defaultValue` fallback
- Don't gate UI on `useLiveQuery` results for click handlers — fetch imperatively instead to avoid race conditions
- DB is currently at version **8** — increment before adding any new indexes/tables
